前言

sonar实际上是由两部分组成的，使用了经典的C/S架构设计，其中扮演客户端（Client端）的是sonar-scanner，这是一个命令行脚本，扮演服务器端（server端）的称为SonarQube，这是一个Web平台，两者统称为Sonar。在本次持续集成中，使用了官网最新的稳定版本7.1。

SonarQube7版本和之前相比，增加了一些新特性：

  • 引入了elasticsearch，elasticsearch是一个全文搜索引擎，详细介绍点我。
  • 代码问题重新分级，将问题分为bug、漏洞、坏味道；将代码检查结果从可靠性、安全性、可维护性几个角度进行问题分类和风险分级；
  • 不再支持原来需要输入系统用户名密码，数据库用户密码的认证方式，统一修改成token方式，参数配置更简便也更安全；
  • 支持webhook功能，可与jenkins等持续集成平台完美对接，在检查完后通过webhook的方式实时反馈检查结果，控制pipeline状态；
  • 更丰富的代码检查规则，更友好的问题处理曲线展示，更清晰的质量阈和代码规则定制。

Sonar简介

SonarQube是一个用于代码质量管理的开源平台，用于管理Java源代码的质量。通过插件机制，SonarQube可以集成不同的测试工具，代码分析工具，以及持续集成工具，比如pmd-cpd、checkstyle、findbugs、Jenkins。通过不同的插件对这些结果进行再加工处理，从而可以方便地对不同规模和种类的工程进行代码质量管理。

启动

启动sonar需要先启动elasticsearch，而elasticsearch只能以普通用户启动，我们在安装sonar过程中都是以root用户来安装的

创建普通用户，并授权

# 创建用户
useradd sonar
 
# 给予权限
chown -R sonar.sonar  /usr/local/sonarqube
 
# 切换用户启动es
su - sonar
# 启动es
/usr/local/sonarqube/elastisearch/bin/elastisearch -d
# 此时会默认在/usr/local/sonarqube/temp/conf下生成es的配置文件
# 默认启动端口为9001
 
# 启动sonar
/usr/local/sonarqube/bin/linux-x86-64/sonar.sh start

参数说明：
    Name：SonarQube
    Server URL：http://IP:9000/sonar （之前配置SoanrQube的地址）
    Server version：根据SonarQube 版本选择
    Server authentication token：在 SonarQube 平台上生成的token值（该值一般在SonarQube初次启动时配置，也可后期在`SonarQube` → `配置` → `权限` → `选择用户` → `管理员` → 点击 admin 用户 `Tokens`里配置）
    SonarQube account login：登录 SonarQube 用户名
    SonarQube account password：登录 SonarQube 密码
    Database URL：之前安装SonarQube时 `sonar.properties` 配置中 `sonar.jdbc.url` 的值
    Database login：数据库用户名
    Database password：数据库密码

另外，新的SonarQube版本不再使用username/password的方式，而是使用token认证的方式，所以上述参数只配置前4项即可。

Pipeline代码

stage ('静态代码扫描') {
    //when指令允许Pipeline根据给定的条件确定是否执行该阶段,isStage1为真时，执行静态代码扫描
    when { expression {return isStage1 } }
    steps{
        echo "**********开始静态代码扫描！**********"
        withSonarQubeEnv('SonarQube') {
            sh "mvn -f pom.xml clean compile sonar:sonar"
        }
        script {
            timeout(120){
                def qg = waitForQualityGate()
                if (qg.status != 'OK') {
                    error "本次扫描未通过Sonarqube的代码质量阈检查，请及时修改！failure: ${qg.status}"
                }
            }
        }
    }
}

fireline

简介

另外，Fireline是360公司开发的一款免费静态代码分析工具。主要针对移动端Android产品进行静态代码分析。其最为突出的优点就是资源泄漏问题的全面检测。同时针对移动端安全漏洞的检测规则。

火线拥有四大类规则，分别为安全类，内存类，日志类，基础类。
• 安全类：根据360信息安全部门最权威的SDL专门定制，每一条SDL都有真实的攻击案例
• 内存类：各种资源关闭类问题检测（本次评测的重点）
• 日志类：检测日志输出敏感信息内容的规则
• 基础类：规范类、代码风格类、复杂度检查规则
（详见官网：http://magic.360.cn）
