前言

每个公司都有自己的环境部署方式和部署脚本，而且大部分情况下都由运维人员来负责，这里仅记录下平时工作持续集成中的部署。

环境介绍

目前主要的环境只有三个：开发环境、内部测试环境、外部测试环境(商户联调环境)，每个项目的module也都没有超过20个，所以只是使用了单机+虚拟服务器的方式部署，还没使用高大上的Doker和K8s集群（眼界的重要啊O(∩_∩)O）

多环境部署

在pipeline中或者jenkins配置Choice Parameter参数target_env，值为环境列表：

parameters {
choice(name:'environment',choices:'dev环境\ntest环境\ntst环境',description: '请选择打包环境')
}

然后从配置文件中读取环境列表和主机列表，分别拉取不同分支的代码并部署到相应的环境。

几个脚本说明如下：

  • env_conf.yml：存储了各编译版本配置以及各个环境的代码分支；
  • ftptool.sh：文件上传与下载；
  • update.sh：上线脚本；
  • target_host.yml：存储了每个应用的环境的主机地址及端口号，类似如下：

stage('部署测试环境') { 
            steps {
                echo "starting deploy to ${serverIP}......"
                //编译和打包
                sh "mvn  -f ${params.pomPath} clean package -Dautoconfig.skip=true -Dmaven.test.skip=true"
                archiveArtifacts warLocation
                script {
                    wrap([$class: 'BuildUser']) {
                    //发布war包到指定服务器，虚拟机文件目录通过shell脚本初始化建立，所以目录是固定的
                    sh "sshpass -p ${serverPasswd} scp ${params.warLocation} ${serverName}@${serverIP}:htdocs/war"
                    //这里增加了一个小功能，在服务器上记录了基本部署信息，方便多人使用一套环境时问题排查，storge in {WORKSPACE}/deploy.log  & remoteServer:htdocs/war
                    Date date = new Date()
                    def deploylog="${date.toString()},${BUILD_USER} use pipeline  '${JOB_NAME}(${BUILD_NUMBER})' deploy branch ${params.repoBranch} to server ${serverIP}"
                    println deploylog
                    sh "echo ${deploylog} >>${WORKSPACE}/deploy.log"
                    sh "sshpass -p ${serverPasswd} scp ${WORKSPACE}/deploy.log ${serverName}@${serverIP}:htdocs/war"
                    //jetty restart，重启jetty
                    sh "sshpass -p ${serverPasswd} ssh ${serverName}@${serverIP} 'bin/jettyrestart.sh' "
                    }
                }
            }
        }

#deploy.sh
# 设置项目位于主干svn的目录
svn_url=*********
# 截取项目svn目录名 - - baseUtil
svn_job_name=`echo ${svn_url##*/}`
# 截取项目父url 
svn_url_parent=`echo ${svn_url%/*}`
# 若项目包含trunk目录, 则其父目录应该再上一层
[[ $svn_url_parent =~ trunk$ ]] && svn_url_parent=${svn_url_parent%/*}
echo "获取到项目父目录: $svn_url_parent"
# 根据选择的参数, 决定 svn地址 / mvn -P / 目标服务器ip
profile=`cat /app/config/jenkins/env_conf.yml | shyaml get-value env.$target_env.profile`
jar_type=`cat /app/config/jenkins/env_conf.yml | shyaml get-value env.$target_env.jar_type`
# 清除上次build代码
rm ./* ./.??* -rf
# svn 检出目标代码
branch_dir=`cat /app/config/jenkins/env_conf.yml | shyaml get-value svn.$jar_type`
if [ "-dev" = "$jar_type" ];then
	echo "主干环境, 将使用dev代码构建"
	svn checkout $svn_url .
elif [ "None" = "$branch_dir" ];then
	echo "未指定分支环境, 将使用dev代码构建"
	svn checkout $svn_url .
else
	svn checkout $svn_url_parent/$branch_dir$svn_job_name .
fi

source /etc/profile
# maven 打包
mvn clean package -P $profile war:war

# =============================== war包专有 ===========================
# 构建后的操作

# 跳板机地址
pub_ip=`cat /app/config/jenkins/target_host.yml | shyaml get-value public.$target_env.host`
# 传输方式
trans_type=`cat /app/config/jenkins/target_host.yml | shyaml get-value public.$target_env.trans`
# 跳板机用户
pub_user=`cat /app/config/jenkins/target_host.yml | shyaml get-value public.$target_env.user`

if [ "None" = "$pub_user" ];then
	pub_user=`whoami`
fi

## 确定war包名称
cd target
war_name_suff=`ls -l *.war | awk '{print $9}'`
war_name=`echo ${war_name_suff%%.war}`
cd ..

if [ "ftp" = "$trans_type" ];then
	echo "$war_name_suff 将以 ftp 形式上传到 $pub_ip"
    # 读取ftp配置信息
    ftp_port=`cat /app/config/jenkins/target_host.yml | shyaml get-value public.$target_env.port`
    ftp_user=`cat /app/config/jenkins/target_host.yml | shyaml get-value public.$target_env.user`
    ftp_pwd=`cat /app/config/jenkins/target_host.yml | shyaml get-value public.$target_env.pwd`
    ftp_dir=`cat /app/config/jenkins/target_host.yml | shyaml get-value public.$target_env.dir`
    
    # 调用脚本上传
    sh /app/config/jenkins/ftptool.sh put $pub_ip $ftp_port $ftp_user $ftp_pwd $ftp_dir `pwd`/target $war_name_suff
    
elif [ "sftp" = "$trans_type" ];then

	if [ "None" = "$pub_ip" ];then
    	echo "$war_name_suff 将拷贝到 本机~/update/release/$target_env/"
        cp target/$war_name_suff ~/update/release/$target_env/
    else
    	echo "$war_name_suff 将以 sftp 形式上传到 $pub_ip:update/release/"
    	cd target
    	scp $war_name_suff $pub_ip:update/release/
    	cd ..
    fi
    
else
	# 确定项目端口号及目标机器ip
    target_ip=`cat /app/config/jenkins/target_host.yml | shyaml get-value $svn_job_name.$target_env.host`
    port=`cat /app/config/jenkins/target_host.yml | shyaml get-value $svn_job_name.$target_env.port`
    
	if [ "None" = "$pub_ip" ];then
    	echo "$war_name_suff 将 部署到 $target_ip:$port"
        cd target
        cp $war_name_suff ~/update/release
        bash /app/config/jenkins/update_center.sh $war_name $port $target_ip
        
    else
		echo "$war_name_suff 将 scp 到 $pub_ip 后部署到 $target_ip:$port"
    	sh /app/config/jenkins/update_ali.sh  $pub_ip `pwd`/target $war_name_suff $pub_user
    	## 向目标机器执行更新
		ssh $pub_user@$pub_ip "bash update_center.sh $war_name $port $target_ip" 
    fi
    
fi
