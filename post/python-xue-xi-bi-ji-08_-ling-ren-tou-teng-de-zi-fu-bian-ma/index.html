ASCII编码

计算机只能处理数字！

若想要处理文本，就要先把文本转换为数字，早期的计算机在设计时采用8个比特bit作为一个字节byte，并且只有127个字符编码，即：大小写英文字母、数字和一些符号，这些字符被编码表被称为ASCII编码，比如大写字母A的编码是65。问题是，早期的这套编码表只有英文，那中文，日文，韩文怎么办？在多语言的文本中，就会出现乱码。并且要想处理中文，一个字节是不够的，至少需要2个字节。所以Unicode出现了。

Unicode编码

Unicode把所有语言都统一到一套编码里，解决了乱码的问题。Unicode编码通常是2个字节表示一个字符。这是虽然乱码的问题解决了，但是又面临一个新的问题：如果文本里全是英文，用Unicode编码比ASCII编码需要多一倍的存储空间，因为Unicode编码通常是2个字节，而ASCII编码只要一个。

所以，出现了可变长的编码的UTF-8编码。

UTF-8编码

UTF-8编码把一个Unicode字符根据不同的数字大小编码成1-6个字节，常用的英文字母被编码成1个字节，汉字通常是3个字节，只有很生僻的字符才会被编码成4-6个字节。如果你要传输的文本包含大量英文字符，用UTF-8编码就能节省空间。

还有一点，ASCII编码实际上可以被看成是UTF-8编码的一部分，所以，大量只支持ASCII编码的历史遗留软件可以在UTF-8编码下继续工作。

同一个字符使用不同编码方式表示如下：

字符 ASCII Unicode UTF-8
A 01000001 00000000 01000001 01000001
中 x 01001110 00101101 11100100 10111000 10101101

计算机的字符编码

在计算机内存中，统一使用Unicode编码，当需要保存到硬盘或者需要传输的时候，就转换为UTF-8编码。用记事本编辑的时候，从文件读取的UTF-8字符被转换为Unicode字符到内存里，编辑完成后，保存的时候再把Unicode转换为UTF-8保存到文件;浏览网页的时候，服务器会把动态生成的Unicode内容转换为UTF-8再传输到浏览器。

Python3中的编码问题

Python 3版本中，字符串是以Unicode编码的，这样就遇到一个问题：上面说过，计算机保存文本到磁盘，或者网络传输时，是用UTF-8编码的，所以，python3写入数据流(文本等)或者在网络上传输时，就需要把str变为以字节为单位的bytes。

以Unicode表示的str通过encode()方法可以编码为指定的bytes;相反，我们从网络或磁盘上读取了字节流，那么读到的数据就是bytes,要把bytes变为str，就需要用decode()方法。
